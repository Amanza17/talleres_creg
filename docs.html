Programa un piloto autonomo de F1

Tu codigo lee una camara frontal, calcula donde esta la linea y decide la potencia de los motores. La fisica avanza cada tick, pero tu controlador solo decide cada varios ticks, como en un sistema real.

Modelo de simulacion

Frecuencia de control y fisica

La simulacion usa ticks fisicos de 0.02 s. Tu programa no recalcula motores en cada tick: mantiene la ultima orden mientras el coche sigue avanzando.

Por defecto, sleep() avanza 3 ticks fisicos. Eso significa que tu controlador decide aproximadamente cada 0.06 s. Puedes usar sleep(5) o sleep(10) para hacerlo mas dificil y simular mas retardo.

Orden de motoresset_motor() no mueve el coche por si solo: deja preparada la potencia para los siguientes ticks.
InerciaLos motores y el giro no cambian instantaneamente. Un KP alto puede llegar tarde y provocar rebotes.
AdherenciaSi pides demasiado giro a demasiada velocidad, aparece derrape y puede terminar en trompo.
API Python

Funciones disponibles

La API por defecto usa nombres estilo Python. No hay wrappers en espanol salvo que importes spanish.py.

camera = read_camera()

Devuelve 11 valores 0/1 de izquierda a derecha. Es la entrada principal para calcular el error.

left, center, right = read_line()

Compatibilidad con tres sensores clasicos. Menos preciso que la camara.

set_motor(left, right)

Potencia de cada lado entre -100 y 100. Si el motor derecho es mayor, el coche gira a la izquierda o derecha segun la diferencia.

sleep(cycles=3)

Avanza la fisica. Tu codigo queda parado mientras el coche sigue moviendose con la ultima orden.

get_speed()

Velocidad actual del coche. Sirve para bajar potencia en curva o detectar inestabilidad.

get_progress()

Progreso de vuelta entre 0 y 1. Util para estrategias por sector.

time()

Tiempo simulado en segundos. Funciona igual en Windows y Linux.

print(...)

Imprime en consola. Usalo cada 25-50 decisiones, no en cada iteracion.

Camara lineal

Como calcular el error

read_camera() devuelve pixeles activos donde la camara ve la linea. El centro de la camara es el pixel medio. El centroide de los pixeles activos te dice hacia donde esta desplazada la linea.

camera = read_camera()
total = sum(camera)
center = (len(camera) - 1) / 2
weighted = 0

for i, value in enumerate(camera):
    weighted += (i - center) * value

error = weighted / total / center

Resultado: error negativo significa linea a la izquierda, error positivo significa linea a la derecha, y 0 significa centrado.

Control P / PI / PID

Que hace cada termino

El controlador convierte el error de camara en una correccion de motor. Esa correccion se resta a un lado y se suma al otro.

P: Proporcional

correction = error * KP

Gira segun el error actual. Si KP es bajo, el coche se abre en curvas. Si es alto, rebota, zigzaguea o trompea.

I: Integral

integral += error correction += integral * KI

Acumula error pasado. Puede corregir sesgos persistentes, pero en este reto suele ser peligroso: si acumula demasiado, tarda en deshacer el giro.

D: Derivativo

derivative = error - last_error correction += derivative * KD

Frena cambios bruscos. Ayuda a que un KP util no provoque rebotes.

Velocidad adaptativa

if abs(error) > 0.35: speed = CORNER_SPEED

No todo se arregla con PID. Si el error crece, baja velocidad antes de subir mucho KP.

Efectos de los parametros

KP bajoGira tarde, se abre y pierde la linea.
KP altoRebota de lado a lado; con la fisica nueva puede hacer trompo.
KD bajoEl coche corrige, pero llega con oscilacion.
KD altoSe vuelve lento de reaccion y puede no cerrar curvas rapidas.
KI altoAcumula memoria excesiva; peligroso si la camara pierde la linea.
Velocidad altaNecesita anticipacion. Baja en curva antes de tocar demasiado el PID.
Plantilla de trabajo

Estructura recomendada

MAX_PASOS = 700
STRAIGHT_SPEED = 16
CORNER_SPEED = 8
KP = 28
KD = 22

last_error = 0
last_side = 0


def calculate_error():
    global last_side
    camera = read_camera()
    total = sum(camera)

    if total == 0:
        return last_side * 1.2

    center = (len(camera) - 1) / 2
    weighted = 0
    for i, value in enumerate(camera):
        weighted += (i - center) * value

    error = weighted / total / center
    if error != 0:
        last_side = -1 if error < 0 else 1
    return error


def choose_speed(error):
    if abs(error) > 0.35:
        return CORNER_SPEED
    return STRAIGHT_SPEED


def control(error):
    global last_error
    derivative = error - last_error
    last_error = error
    return error * KP + derivative * KD


for step in range(MAX_PASOS):
    error = calculate_error()
    speed = choose_speed(error)
    correction = control(error)
    set_motor(speed - correction, speed + correction)
    sleep()
Compatibilidad

Windows y Linux

El servidor evita APIs exclusivas de Unix como signal.alarm. El tiempo limite de Python se controla con trazado interno y time.monotonic(), disponible en Windows y Linux.

La API por defecto usa nombres Python. Si alguien necesita alias en espanol puede escribir from spanish import *.
